### 1. REDEREF — Training-Free мультиагентная координация для LLM

**Источник:** arXiv (2603.13256)

**Описание:**  
REDEREF — это фреймворк для вероятностного управления мультиагентными LLM-системами, который не требует дообучения. Использует механизм belief-guided делегирования через Thompson sampling, рефлексивную маршрутизацию с калиброванным судьёй, и агентный выбор на основе доказательств вместо простого усреднения результатов. Эксперименты показывают сокращение использования токенов на 28%, вызовов агентов на 17% и времени успеха на 19% по сравнению со случайным делегированием.

**Почему важно:**  
Первый практичный фреймворк координации, который можно внедрить в любую существующую систему без переобучения — это решает главную проблему масштабирования мультиагентов.

**Применение для AstroFinSentinelV5:**  
REDEREF может быть использован для оптимизации маршрутизации задач между агентами в AstroFinSentinelV5 — например, для динамического распределения финансовых запросов между специализированными агентами мониторинга, анализа и алертинга с автоматическим перенаправлением при низкой уверенности.

### 2. Orloj — Open-Source оркестрация мультиагентов через YAML

**Источник:** GitHub (OrlojHQ/orloj)

**Описание:**  
Orloj — это production-grade платформа для оркестрации мультиагентных AI-систем с declarative YAML конфигурацией. Поддерживает сложные топологии (pipeline, hierarchical, swarm), маршрутизацию между моделями (OpenAI, Anthropic, Azure, Ollama), изоляцию инструментов в контейнерах/sandbox-ах, политики governance (RBAC, token caps), retries и dead-letter handling. Активно развивается — недавний релиз v0.6.0 с нативными инструментами для кросс-тасковой оркестрации.

**Почему важно:**  
Orloj предоставляет enterprise-уровня управление мультиагентами с version-controlled конфигурациями и inline политиками — то, чего обычно не хватает в self-hosted решениях. Это-ready для перехода от прототипов к продакшену.

**Применение для AstroFinSentinelV5:**  
Orloj может стать базовой инфраструктурой для AstroFinSentinelV5 — заменить ad-hoc скрипты оркестрации на declarative YAML, что упростит управление агентами, добавит governance и observability из коробки.

### 3. GitHub Squad — Мультиагентные команды внутри репозитория

**Источник:** GitHub (bradygaster/squad)

**Описание:**  
Squad — это open-source фреймворк от GitHub (альфа, но активно развивается), который позволяет создавать AI-команды внутри проекта через GitHub Copilot. Координатор динамически спавнит специализированных агентов (lead, frontend, backend, tester), которые работают параллельно с общим контекстом репозитория. Недавно вышел `/fleet` command для параллельного запуска нескольких агентов одновременно.

**Почему важно:**  
Squad демократизирует мультиагентную разработку — вместо сложной инфраструктуры достаточно двух команд. Особенно интересна модель с разделением execution/review, где агенты не могут пересматривать свою же работу — это архитектурное решение повышает качество и предсказуемость.

**Применение для AstroFinSentinelV5:**  
Принцип разделения ролей и итеративного ревью из Squad может быть адаптирован для AstroFinSentinelV5: агент-исполнитель генерирует алерт, агент-ревьюер проверяет его корректность перед отправкой — это снизит ложные срабатывания.
