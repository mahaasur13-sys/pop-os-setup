## System Type

**Semantic Distributed State Machine**

Distributed nodes reach consensus on entity state transitions.
Each transition is recorded as a typed `Event` with cryptographic identity.
The `SemanticProjection` provides a consistent view of all events sharing an `entity_hash`.

### v9.10 — Semantic Consistency Lock Layer (current)

Canonical event model that unifies the distributed protocol stack into a formally
consistent semantic execution graph.

**Modules:**
- `federation/semantic/v910.py` — core event model
- `federation/semantic/test_v910.py` — 24 tests, all passing

**Components:**
| Component | Responsibility |
|-----------|---------------|
| `Event` | Immutable semantic record: `(event_id, type, entity_hash, hash_mode, ...)` |
| `EventStore` | In-memory emission, indexing by entity_hash and type |
| `SemanticProjection` | Per-entity canonical view across all event types |
| `SemanticBinder` | Helpers to emit typed events from protocol layer |
| `DriftDetector` | Scans all projections for invariant violations |
| `DriftReport` | Structured report of cross-layer mismatches |

**Semantic Identity Contract:**
```
semantic_id(event) = (event.type, event.entity_hash, event.hash_mode)
```

Two events are the same semantic entity iff their `semantic_id` tuples match.

---

## Design Invariants

1. `Event.event_id` is deterministically derived from content — same content = same ID
2. `EventStore` is append-only within a session; `reset()` clears for test isolation
3. `SemanticProjection.canonical` prefers CONSENSUS > GOSSIP > PROOF > REPLAY > TRUST by timestamp
4. `DriftDetector` scans across all entity hashes; cross-reference checks detect mismatches
5. All 5 event types are independently queryable per entity_hash

## Known Limitations

1. `EventStore` is in-memory only — no persistence across process restarts
2. `DriftDetector._check_gossip_consensus` is a no-op by design (single-projection view cannot detect cross-projection drift)
3. SBS tests require `atomos_pkg` pip package on `sys.path` — handled via `sbs/tests/conftest.py`
