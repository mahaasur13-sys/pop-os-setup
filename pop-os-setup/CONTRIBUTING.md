# Contributing to pop-os-setup

Thank you! This is an autonomous Linux deployment agent for Pop!_OS 24.04 NVIDIA.

## 🎯 What This Project Does

Automated 26-stage setup for AI/Dev workstations:
- NVIDIA GPU drivers + CUDA
- KDE Plasma desktop
- Docker / Kubernetes (k3s)
- Slurm HPC cluster
- Ray distributed compute
- Ceph storage
- AI/ML tooling stack

## 🛠️ Dev Environment

```bash
git clone https://github.com/mahaasur13-sys/pop-os-setup.git
cd pop-os-setup

# Dry-run (Stage 1 only, no changes)
./pop-os-setup-v5.sh --stage 1 --profile ai-dev --dry-run
```

## ✅ PR Checklist

Before opening a PR:

- [ ] `./pop-os-setup-v5.sh --dry-run` completes all 26 stages without error
- [ ] No stages broken by dependencies
- [ ] `--profile` flags work: `workstation`, `cluster`, `ai-dev`, `full`
- [ ] `--skip-*` flags correctly bypass skipped stages
- [ ] idempotent — running twice doesn't break anything
- [ ] README.md Stage table updated if adding/removing stages

## 🔒 Safety Rules

- **NEVER** modify Stages 1–3 without explicit user confirmation
- **ALWAYS** test in VM before pushing (use `--dry-run` or VM snapshot)
- **NEVER** commit credentials or host-specific configs
- Stages modifying `/etc`, `/boot`, `crypttab` require root — scripts already use `sudo`

## 🧪 Testing

```bash
# Full dry-run (no changes applied)
./pop-os-setup-v5.sh --profile ai-dev --dry-run

# Specific stage range
./pop-os-setup-v5.sh --profile workstation --from 10 --to 15

# Skip GPU stages (CPU-only)
./pop-os-setup-v5.sh --profile workstation --skip-cuda --skip-docker

# Individual stage
./pop-os-setup-v5.sh --stage 20
```

## 🔧 Adding a New Stage

1. Assign next available stage number
2. Add function `stage_N_name()` in `pop-os-setup-v5.sh`
3. Update README.md Stage table
4. Add to `main()` in correct position
5. Document failure recovery in section header comment
6. Test: `./pop-os-setup-v5.sh --stage N --dry-run`

## 🔄 Release Process

1. Update `pop-os-setup-v5.sh` (stable) or `pop-os-setup-v4.sh` (legacy)
2. Update `RELEASE_NOTES.md`
3. Tag: `git tag vX.Y.Z && git push --tags`
4. SLSA attestation bundle auto-generated by CI

## ⚠️ Common Issues

| Problem | Fix |
|---------|-----|
| `systemd-boot` fails | Boot in UEFI mode, not Legacy |
| NVIDIA driver black screen | Use `nomodeset` boot param, then reinstall |
| Disk encryption passphrase rejected | Use ASCII-only, ≥20 chars |
| k3s install hangs | Check network connectivity, try `--disable=traefik` |
