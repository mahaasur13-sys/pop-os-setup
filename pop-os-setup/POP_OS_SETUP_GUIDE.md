## ⚠️ Что это и зачем

Этот гайд поможет установить и настроить **Pop!_OS 24.04 LTS NVIDIA Edition** на твоём компьютере. Pop!_OS — это Linux-дистрибутив, оптимизированный для GPU (в том числе NVIDIA) и разработчиков.

**Два варианта использования:**

| Вариант | Что делает | Кому подходит |
|---------|------------|---------------|
| **Только GUI-гайд** | Установка вручную, без скрипта | Хочешь понять процесс сам |
| **GUI-гайд + pop-os-setup** | Автоматическая настройка Dev/AI tools | Хочешь быстро и безопасно |

---

## 📋 Что понадобится

```
✅ Компьютер с NVIDIA GPU (или без GPU — скрипт определит)
✅ USB-флешка минимум 8 ГБ
✅ Интернет-соединение (для скачивания ~3 ГБ)
✅ 30–60 минут свободного времени
✅ Резервная копия важных данных (на всякий случай)
```

### Проверка, что у тебя NVIDIA

Открой терминал в текущей системе (Windows → Win+R → `cmd` → `powershell`):

```powershell
wmic path win32_VideoController get name
```

Или просто посмотри на коробку видеокарты. Если NVIDIA указана — ты по адресу.

## 🟠 Шаг 1 — Скачать Pop!_OS

### 1.1 Перейди на официальный сайт

Открой в браузере: **https://pop.system76.com**

Нажми **"Download"** → выбери **Pop!_OS 24.04 LTS**

### 1.2 Выбери редакцию

| Редакция | Когда выбирать |
|----------|---------------|
| **NVIDIA Edition** | Если у тебя NVIDIA видеокарта (RTX/GTX серии) |
| **Intel/AMD Edition** | Если видеокарта AMD, Intel или без дискретной GPU |

> ⚠️ **Важно:** Если сомневаешься — бери **NVIDIA Edition**. Она работает и без NVIDIA GPU (просто драйвер не установится).

### 1.3 Скачай ISO-файл

- Файл называется примерно: `pop-os_24.04_amd64_nvidia_7.iso`
- Размер: **~3 ГБ** — нужен хороший интернет
- Проверь, что скачалось: файл должен оканчиваться на `.iso`

### 1.4 Проверка контрольной суммы (рекомендуется)

После скачивания открой терминал в папке с файлом и выполни:

```bash
# Windows (PowerShell)
Get-FileHash pop-os_24.04_amd64_nvidia_7.iso -Algorithm SHA256

# macOS / Linux
shasum -a 256 pop-os_24.04_amd64_nvidia_7.iso
```

Сравни результат с хешем на [официальной странице](https://pop.system76.com).

## 🔧 Шаг 2 — Записать USB и установить систему

### 2.1 Скачай Rufus (для записи на USB)

**Windows:**
- Скачай [R Rufus](https://rufus.ie) (бесплатная программа)
- Установи или запусти портативную версию

**macOS:**
- Используй встроенную утилиту или [Balena Etcher](https://etcher.balena.io)

### 2.2 Запиши ISO на USB

1. Вставь USB-флешку (8 ГБ+)
2. В Rufus выбери:
   - **"Устройство"** → твоя флешка
   - **"Выберите"** → скачанный `.iso` файл
   - **"Схема раздела"** → GPT
   - **"Целевая система"** → UEFI (или BIOS + UEFI если ставишь на старый ПК)
   - **"Файловая система"** → FAT32
3. Нажми **"СТАРТ"**
4. Подтверди предупреждение (все данные на флешке будут стёрты)

> ⏱️ Запись занимает 5–15 минут в зависимости от скорости USB.

### 2.3 Загрузись с USB

1. **Выключи компьютер**
2. **Вставь USB-флешку**
3. **Включи и сразу нажимай** клавишу для Boot Menu:
   - Обычно `F12`, `F8`, `F11`, `Esc` или `Del`
   - Какая именно — зависит от производителя (ASUS, MSI, Gigabyte и т.д.)
   - Если не знаешь — гугли: `"[модель материнской платы]" boot menu key`

4. В меню выбери свою флешку (обычно указана как USB или имя флешки)
5. Нажми Enter

### 2.4 Установка Pop!_OS

После загрузки с USB появится рабочий стол Pop!_OS (live-режим, ничего не установлено).

1. На рабочем столе дважды кликни **"Install Pop!_OS 24.04 LTS"**
2. **Выбери язык:** Русский (или English)
3. **Раскладка клавиатуры:** Русская (или оставь English)
4. **Тип установки:**

| Вариант | Что делает | Риск |
|---------|-----------|------|
| **"Очистить диск и установить Pop!_OS"** | Удаляет всё, ставит Pop!_OS | ⚠️ Все данные стираются |
| **"Настроить разделы вручную"** | Для опытных | ⚠️ Можно сломать |
| **"Стереть и установить рядом с Windows"** | Dualboot | ⚠️ Средний |

> 🚨 **Если не уверен — выбери первый вариант, но убедись что нет важных данных на диске.**

5. **Введи данные пользователя:**
   - Имя: `ТвоёИмя`
   - Имя компьютера: `pop-os` (или любое)
   - Пароль: `ПридумайНадёжныйПароль`
   - ⚠️ **Не забудь пароль!**

6. Нажми **"Начать установку"**
7. ⏱️ Подожди **10–20 минут**
8. После завершения нажми **"Перезагрузить сейчас"** (вытащи USB сразу после выключения)

## 🔄 Шаг 3 — Первая загрузка и настройка BIOS

### 3.1 Первая загрузка

После перезагрузки:
1. Должен появиться логин Pop!_OS
2. Введи логин и пароль, которые указал при установке
3. Если видишь рабочий стол — установка прошла успешно ✅

### 3.2 Настройка NVIDIA (если есть GPU)

Если у тебя NVIDIA:
1. Открой терминал: `Ctrl + Alt + T`
2. Выполни:

```bash
nvidia-smi
```

Должна появиться таблица с информацией о GPU. Если видишь "No NVIDIA GPU" — значит драйвер ещё не установлен.

3. Установи драйвер NVIDIA:

```bash
sudo system76-driver nvidia
```

Или через графический интерфейс:
- **System Settings → Software & Updates → Additional Drivers**
- Выбери "NVIDIA driver (proprietary)"
- Нажми "Apply Changes"
- Перезагрузись

### 3.3 Подключись к интернету

Убедись, что Wi-Fi подключён (иконка в правом верхнем углу).

## ⚡ Шаг 4 — Установка pop-os-setup

> ⏱️ Этот шаг автоматизирует настройку Dev/AI окружения.

### 4.1 Открой терминал

Нажми `Ctrl + Alt + T` или кликни на иконку терминала.

### 4.2 Скачай скрипт

```bash
# Вариант А — скачать с GitHub Release (самый надёжный)
curl -fsSL https://github.com/mahaasur13-sys/pop-os-setup/releases/download/v3.0.0/pop-os-setup.sh -o pop-os-setup.sh

# Вариант Б — скачать напрямую с raw.githubusercontent.com (может быть 404 из-за CDN-задержки)
curl -fsSL https://raw.githubusercontent.com/mahaasur13-sys/pop-os-setup/6dab53a/pop-os-setup.sh -o pop-os-setup.sh

# Делаем исполняемым
chmod +x pop-os-setup.sh
```

> 💡 Вариант А всегда работает — файлы в Release не зависят от CDN. Вариант Б может возвращать 404 первые 5–15 минут после пуша.

### 4.3 Проверь, что скачалось

```bash
# Должно вывести "Pop!_OS Setup v3.0.0"
./pop-os-setup.sh --version
```

Если видишь ошибку — проверь интернет-соединение.

### 4.4 Что делает скрипт

| Профиль | Что устанавливает |
|---------|-------------------|
| `workstation` | Zsh, Docker, Python, Neovim, базовые инструменты |
| `ai-dev` | Workstation + CUDA, Ollama, Jupyter, PyTorch |
| `cluster` | Workstation + k3s, Longhorn, MetalLB, Ceph |
| `full` | Всё перечисленное выше |

## 🎯 Шаг 5 — Выбор профиля и запуск

### 5.1 Интерактивный режим (самый простой)

```bash
sudo bash pop-os-setup.sh
```

Скрипт задаст вопросы и предложит выбрать профиль.

### 5.2 Режим без вопросов (для опытных)

```bash
# Только workstation (самый быстрый)
sudo PROFILE=workstation bash pop-os-setup.sh

# Полная установка (всё включая AI/ML)
sudo PROFILE=full bash pop-os-setup.sh

# Только AI-инструменты
sudo PROFILE=ai-dev bash pop-os-setup.sh
```

### 5.3 Пробный запуск (без изменений)

```bash
sudo bash pop-os-setup.sh --dry-run
```

Покажет, что **будет** сделано, но ничего не изменит.

### 5.4 Запуск одной стадии

```bash
# Только stage 5 (Zsh)
sudo bash pop-os-setup.sh --stage 5

# Пропустить stage 10 (hardening)
sudo bash pop-os-setup.sh --skip-stage 10
```

### 5.5 Что происходит во время установки

Скрипт выводит информацию в терминал:

```

   ✓ zsh package installed
   ✓ Cloned: ohmyzsh
   ✓ Oh My Zsh + plugins installed

   Configuring UFW firewall (default deny incoming)...
   ✓ UFW enabled
   Configuring fail2ban...
   ✓ fail2ban configured

🔧 [26/26] FINAL REPORT
```

⏱️ **Полная установка (профиль `full`):** 20–40 минут в зависимости от скорости интернета.

## ✅ Шаг 6 — Проверка после установки

После завершения работы скрипта выполни команды для проверки:

### GPU

```bash
nvidia-smi
```

Ожидаемый результат: таблица с GPU, driver version, CUDA version.

### Docker

```bash
docker ps
```

Ожидаемый результат: список контейнеров (может быть пустой, это нормально).

### kubectl (если выбран cluster/ai-dev)

```bash
kubectl get nodes
```

Ожидаемый результат: `No resources found` (если k3s ещё не настроен) или `Ready`.

### Zsh

```bash
zsh --version
```

### Neovim

```bash
nvim --version
```

### Fail2ban

```bash
sudo systemctl status fail2ban
```

Должен показывать `active (exited)` или `active (running)`.

## 🔧 Шаг 7 — Что делать если что-то пошло не так

### Проблема: Скрипт завис

- Нажми `Ctrl + C` для отмены
- Запусти снова — скрипт **идемпотентный** (безопасна повторная работа)

### Проблема: Нет интернета

```bash
# Проверь Wi-Fi
nmcli device status

# Подключись к Wi-Fi
nmcli device wifi connect "ИМЯ_СЕТИ" password "ПАРОЛЬ"
```

### Проблема: NVIDIA не работает после установки

```bash
# Переустанови драйвер
sudo apt install --reinstall nvidia-driver-550
sudo reboot
```

### Проблема: Docker не запускается

```bash
# Проверь статус
sudo systemctl status docker

# Включи и запусти
sudo systemctl enable docker
sudo systemctl start docker
```

### Проблема: k3s не устанавливается

```bash
# Проверь, включён ли swap
swapon --show

# Если swap выключен — включи
sudo fallocate -l 2G /swapfile
sudo chmod 600 /swapfile
sudo mkswap /swapfile
sudo swapon /swapfile
```

### Проблема: Ошибка "command not found"

Обнови терминал или открой новый:

```bash
# Перезагрузи shell
exec zsh
# или
exec bash
```

### Проблема: Хочешь начать сначала

Скрипт безопасен для повторного запуска. Но если хочешь чистый лист:

```bash
# Удали конфигурационные файлы
rm -rf ~/.config/pop-os-setup
rm -rf ~/.config/nvim

# Перезапусти скрипт
sudo bash pop-os-setup.sh
```
